# Exit Simulation on REAL Live Plays — 2026-07-13

**Status:** RESEARCH ONLY (Clause 0). No live-code change is proposed as shipped.
The one robust candidate below goes to **DECISIONS NEEDED**, not to the tracker.

## Question

Does ANY exit family beat the tracker's CURRENT structure-invalidation exit's
**realized** P&L, **out-of-sample**, at small sample? Saturday's exit-study
(1,295 replayed fires) already found naive trailing exits did **not** beat
baseline OOS. Here we test on the **173 real live plays** (4 days, 2026-07-08 →
07-13) using **actual per-minute option marks** pulled from UW
`/option-contract/{occ}/intraday`.

## Pre-registered method (written before viewing aggregates)

- **Entry anchor** = `entry_mark` (recorded fill) = denominator for every return.
- **Price series** = UW intraday per-minute `close`, from the fire minute →
  session close. **All exits are close-basis** (evaluate each minute's close; no
  intra-bar look-ahead). A rule that never triggers exits at session close.
- **Baselines** (what we must beat):
  - **DB baseline** = `(close_mark − entry_mark)/entry_mark` — actual realized in
    the tracker.
  - **Path baseline** = the path mark at `close_ts_ms`, relative to `entry_mark` —
    the current exit **reconstructed on the same price series as the sims** →
    apples-to-apples. DB baseline reported for context; the head-to-head uses the
    **path** baseline. (They agree closely: −20.2% DB vs −21.1% path, distinct
    fires — the reconstruction is faithful.)
- **Families:** PT +25/+50/+75/+100; TRAIL 20/30/40 (off running peak); hard STOP
  −30/−50; TIME 30/60/90 min; SCALE (½ @ +50, trail rest 30%); LOCK (BE after
  +30, then trail 30); LOCK (+25 after +75, trail 20); plus FLOOR (hold EOD).
- **Dedupe:** 7/08 fired **4 strikes per (fire, ticker, state)**; representative =
  strike closest to `spot_at_fire` (matches the 1-contract logic of later days).
  **Primary aggregation = distinct fires, equal-weight.** Also reported:
  per-contract (all 167 with paths) and day-balanced mean.
- **Walk-forward:** train = 7/08+7/09, test = 7/10+7/13. A family "passes" only if
  Δ-vs-path-baseline > 0 on **both** splits.
- **Robustness:** leave-one-day-out (LOO) — worst Δ across dropping each day.

### Data quality notes
- 167/170 non-null-close plays had a usable UW path (3 missing). Distinct fires =
  **66** (7/08: 36 · 7/09: 5 · 7/10: 11 · 7/13: 14).
- `entry_mark` vs first path mark: **median abs diff 2.0%**, mean 5.6% — a few
  wide-gap contracts, otherwise tight.
- **Baseline exit type differs by day.** 7/08's baseline is the *older*
  `eod`/`state_clear` logic; **7/10 + 7/13 are almost entirely
  `structure_invalidated`** (the current exit). So the walk-forward **test split
  (7/10+7/13) is the clean test of "beat the CURRENT exit."**

## Result 1 — the leak is real and asymmetric

Distinct fires: **path-baseline avg −21.1%** while **MFE (path peak) avg +72.5%**.
Winners round-trip. Decomposition: **27 of 66 fires realize worse than −50%**
under the current exit — it **holds losers into deep 0DTE decay** *and* exits
winners early. Beating a baseline this mis-timed is therefore a **low bar**, and
~14/16 families clear it in-sample. The interesting question is which survive
**robustness + walk-forward**, and by a margin beyond cost/mid-noise (~2–3%).

## Result 3 — only ONE family is robust across every cut

Requiring **LOO-worst Δ > cost/noise band (~+3%)** AND **walk-forward pass**:

- **STOP-30 stands alone.** LOO-worst **+12.2%** (every day-drop keeps it +12→+20);
  walk-forward train **+14.7%**, test **+20.6%**. It is the only family whose edge
  doesn't evaporate when any single day is removed.
- **Profit-targets (PT+75 +4.6, PT+25 +5.5) are modestly robust** but their margin
  (+3–5%) sits near the cost/mid-noise band → treat as weak.
- **Trailing stops are NOT robust** — TRAIL20/30 flip negative when 7/08 is
  dropped (they are 7/08-fit); TRAIL20 fails the test split. **This reproduces
  Saturday's negative result on trailing exits.**
- **FLOOR (hold-to-EOD) "beats" baseline on average (+3.4)** but with median
  −76.5% and LOO-worst −5.6% — it's a **trend-day tail artifact** (7/09, 7/10,
  7/13 had trend legs), not a rule.

### How STOP-30 actually wins — read this before believing it
It is a **loss-cutter, not a winner-capturer.** It **reduces winners 23 → 11** (it
stops out of fires that would have recovered) but **eliminates the deep-decay tail
entirely: 27 baseline losses worse than −50% → 0.** Its whole edge is
**truncating the left tail** the current exit walks into. That is intuitive and
un-p-hacked, but it is *risk management*, **not signal alpha.** It says "the
current exit is worse than a dumb −30% stop," not "we found an edge."

## Verdict

**One candidate survives — a hard stop-loss near −30% — but it clears a
genuinely mis-timed baseline, wins by cutting losers (not capturing winners),
and rests on 66 distinct fires (25 out-of-sample). CANDIDATE for forward
validation, NOT a change to ship.** Naive trailing exits again fail OOS,
consistent with Saturday.

### Honest caveats (mandatory)
1. **n is tiny and lumpy.** 66 distinct fires; 36 are 7/08 (older exit logic);
   the OOS test is **25 fires over 2 days**, and 7/13 is swung by a couple of
   +100%+ movers. **Directional, not conclusive.**
2. **The baseline is a low bar.** The current structure-invalidation exit holds
   losers into deep decay *and* cuts winners early — ~14/16 families beat it
   in-sample. "Beats baseline" here ≈ "the current exit is poorly timed," not "we
   discovered a good exit."
3. **Costs largely cancel in the delta** (every single-exit family incurs one exit
   spread, same as baseline) **except** stop-fills: hard stops in fast 0DTE decay
   **fill worse than the trigger close**, so STOP-30's edge is somewhat
   optimistic. Its +12–17% margin has cushion; the +3–5% profit-target margins do
   **not** clear costs + slippage confidently.
4. **Mid-vs-trade noise.** UW `close` marks are trade prints (bid- or ask-side),
   not mids — the same illusion Saturday flagged. Treat any Δ < ~3% as zero.
5. **Multiple comparisons.** 16 families × several metrics. The both-splits +
   LOO-worst filter is the discount; only STOP-30 clears it decisively — which is
   *why* it is credible relative to the rest.

### DECISIONS NEEDED (not shipped)
- **D1 — Add a hard −30% stop-loss to the tracker's exit logic, in ADDITION to
  structure-invalidation (whichever triggers first)?** Robust in this study, but
  validate on forward fires first; expect it to raise realized P&L by truncating
  the deep-loss tail while lowering win rate. Do **not** replace
  structure-invalidation with trailing (trailing failed OOS).
- **D2 — Log realized-vs-peak per play going forward** so the −30%-stop candidate
  can be A/B'd on live fires (paper the stop alongside the live exit) before any
  code change.
